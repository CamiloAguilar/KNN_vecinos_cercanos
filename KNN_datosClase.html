KNN - Vecinos más próximos¶

Este es un mtodo de clasificacin no paramtrico, que estima el valor de la funcin de densidad de probabilidad o directamente la probabilidad a posteriori de que un elemento $x$ pertenezca a la clase $C_{j}$ a partir de la informacin proporcionada por el conjunto de prototipos. En el proceso de aprendizaje no se hace ninguna suposicin acerca de la distribucin de las variables predictoras. En el reconocimiento de patrones, el algoritmo $k-nn$ es usado como mtodo de clasificacin de objetos (elementos) basado en un entrenamiento mediante ejemplos cercanos en el espacio de los elementos.

El presente notebook aplicar este concepto sobre los datos artificiales de la clase para estimar Sexo a partir de Peso y Estatura.

La intensin ahora es generar puntos de forma aleatoria dentro de los lmites del grfico anterior, buscando etiquetar cada uno de ellos como masculino o femenino, dependiendo de su cercana a $k$ de los puntos ya conocidos. La definicin de la etiqueta estar dada por la clase del grupo mayoritario dentro de los $k$ vecinos seleccionados, razn por la cual ser necesario siempre escoger una cantidad impar de vecinos, evitando as caer en empates (sin grupo mayoritario).

Para ello resulta necesario definir la siguiente funcin generadora de puntos:

In [4]:
## Lmites de simulacin
lims <- c(1, 2.5,#Estatura, eje X
          0, 1.5) #Peso, eje Y

## Funcin generadora de puntos
nuevopunto <- function(lims=lims){
  a <- runif(1,lims[1],lims[2])
  b <- runif(1,lims[3],lims[4])
  return(c(a, b))
}

Debido a que nuestros datos de estudio nicamente contenen dos sujetos de sexo feminino, nuestra nica opcin ser definir $k=3$.

El siguiente algoritmo realiza la clasificacin de cada punto (generado de forma aleatoria por la funcin anterior), al encontrar los $k=3$ vecinos mas cercanos y con ellos determinar el grupo con la clase predominante, asignndosela finalmente al punto. En este caso particular se han generado 20000 puntos, donde la regin negra representa todos los puntos que fueron clasificados como de sexo masculino, y la gris como femenino.

In [5]:
## Grfico
plot(X$Estatura, X$Peso, type = "n", xlab="Estatura", ylab="Peso",asp=1,
xlim=c(1,2.5),ylim=c(0,1.5))

## K-vecinos a considerar
k <- 3

## Algoritmo de clasificacin
for(iter in 1:20000){
  x <- nuevopunto(lims)
  diss <- dist(rbind(x,X[,c(2,3)]))[1:21]
  knn <- head(order(diss),k)
  etiqueta <- names(sort(table(X$G[knn]),decreasing = TRUE))[1]
  clase <- as.numeric(etiqueta=="M")
  for(inn in 1:k){
    if(X$G[knn[inn]]==etiqueta){
      points(x[1],x[2],col=c("gray","black")[clase+1],pch=20)
    }
  }
}

## Datos de entrenamiento
cols2 <- rep("red",nrow(X))
cols2[which(X$G=="M")] <- "green"
points(X$Estatura, X$Peso, pch=16, col=cols2)

Es importante notar que al menos para este problema, el algoritmo $k-nn$ resulta ser un muy buen clasificador; tanto puntos verdes (hombres) como rojos (mujeres) quedaron bien clasificados.
